# I7 手腕任务精简：移除“跟随光点快速点击”子任务（2026-07-03）

> 回答需求 3：删除 App「任务」部分 **I7** 中的「跟随光点快速点击」任务，只保留其中的**三种手腕相关动作**。
> 本文记录动机、改动点、不变量与测试结果。（改动在 App 仓库 `ContextAuthlab`；本文放在服务端 `docs/0703` 便于与本轮排查一并归档。）

---

## 1. 结论速览

- **改动**：`I7 手腕转动类` 原本融合了两部分——(a) 三种标准化手腕/手机动作（**左右平移 / 左右转动 / 前后摆动**，由 `WristGuide` 引导）与 (b) 一个 **`跟随光点快速点击` 高频点击画布（`MotionTapCanvas`）**。现**移除 (b)**，`I7` 收敛为**纯手腕转动**（仅保留三种动作）。
- **任务总数不变**：仍为 **8 类 `I0..I7`**——只删 I7 的一个**子任务**，不删 I7 本身。
- **测试**：`:android-app:testDebugUnitTest` 全绿（`LocalizationTest` 仍断言任务数 = 8）；`assembleDebug` 成功，已重新产出 APK。

---

## 2. 动机

- 需求明确：I7 只保留三种手腕相关动作。
- 科学上更纯：I7 定义的核心信号是**手腕运动的高 IMU 能量**；叠加“高频点击画布”会引入**触控节奏这一混杂信号**，且高频点击与 `I2 离散触控`、`I5 对象操控` 的触控证据纠缠。移除后 I7 的 IMU 特征更干净、与其它场景区分度更高。
- 触控证据不丢失：全局 `MainActivity.dispatchTouchEvent → TouchInteractionRecorder` 的触控起止计时**保留**，其它需要触控的任务（如 `I5`）不受影响。

## 4. 不变量与影响面（确认无回归）

- **任务枚举仍 `I0..I7`（8 类）**：`when(task)` 仍 8 分支且无 `else`（穷尽）；`TaskCategory.entries.size == 8`；进度环/卡片按 8 计。
- **上传契约无影响**：批次与 `context_features` 的 task 字段仍同源自 `TaskCategory` 枚举；`task_id==task_category`、`task_sequence==int(task_id[1:])` 对 I7 依旧成立。服务端**无需任何改动**即接受精简后的 I7 批次。
- **数据形态变化（预期内）**：今后 I7 批次的 `touch_events` 通常为 0 / 很少（被试做手腕动作、不点屏幕），`diagnostics.touch_event_count` 与之一致，**不违反任何契约**。
- **未改动的相邻实现**：`WristGuideAnimation.kt`（三种手腕动作示意，正是要保留的三种任务）原样保留；`I5 ObjectManipulationTask` 的批注画布原样保留；`ContextFeatureExtractor` 的 `estimatedContextCategory` 8 类映射不变。
- **导入清理**：`MotionTapCanvas` 用到的 `LocalHapticFeedback / HapticFeedbackType / Random / mutableIntStateOf` 等在文件他处仍在使用，删除后**无悬空导入**（且项目未开启 `allWarningsAsErrors`）。
